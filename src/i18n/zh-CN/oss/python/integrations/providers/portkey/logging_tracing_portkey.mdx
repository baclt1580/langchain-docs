当使用 LangChain 构建应用或代理时，您最终会进行多次 API 调用来满足单个用户请求。然而，当您想要分析这些请求时，它们并不是串联在一起的。通过 [**Portkey**](/oss/integrations/providers/portkey/)，来自单个用户请求的所有嵌入、补全和其他请求都将记录并追踪到同一个 ID，使您能够全面了解用户交互。

本笔记本作为分步指南，介绍如何在您的 LangChain 应用中针对 LangChain LLM 调用使用 `Portkey` 进行日志记录、追踪和监控。

首先，让我们导入 Portkey、OpenAI 和 Agent 工具

```python
import os

from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_openai import ChatOpenAI
from portkey_ai import PORTKEY_GATEWAY_URL, createHeaders
```

在下方粘贴您的 OpenAI API 密钥。 [(您可以在此处找到)](https://platform.openai.com/account/api-keys)

```python
os.environ["OPENAI_API_KEY"] = "..."
```

## 获取 Portkey API 密钥

1. 在此 [注册 Portkey](https://app.portkey.ai/signup)
2. 在您的 [仪表板](https://app.portkey.ai/) 上，点击左下角的个人资料图标，然后点击“复制 API 密钥”
3. 将其粘贴到下方

```python
PORTKEY_API_KEY = "..."  # Paste your Portkey API Key here
```

## 设置追踪 ID

1. 在下方为您的请求设置追踪 ID
2. 追踪 ID 可以是源自单个请求的所有 API 调用的公共 ID

```python
TRACE_ID = "uuid-trace-id"  # Set trace id here
```

## 生成 Portkey 头部

```python
portkey_headers = createHeaders(
    api_key=PORTKEY_API_KEY, provider="openai", trace_id=TRACE_ID
)
```

定义要使用的提示词和工具

```python
from langchain_classic import hub
from langchain.tools import tool

prompt = hub.pull("hwchase17/openai-tools-agent")


@tool
def multiply(first_int: int, second_int: int) -> int:
    """Multiply two integers together."""
    return first_int * second_int


@tool
def exponentiate(base: int, exponent: int) -> int:
    "Exponentiate the base to the exponent power."
    return base**exponent


tools = [multiply, exponentiate]
```

像往常一样运行您的代理。**唯一**的变化是我们将**包含上述头部**在请求中。

```python
model = ChatOpenAI(
    base_url=PORTKEY_GATEWAY_URL, default_headers=portkey_headers, temperature=0
)

# Construct the OpenAI Tools agent
agent = create_openai_tools_agent(model, tools, prompt)

# Create an agent executor by passing in the agent and tools
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

agent_executor.invoke(
    {
        "input": "Take 3 to the fifth power and multiply that by thirty six, then square the result"
    }
)
```

## Portkey 上的日志记录和追踪工作原理

**日志记录**

- 通过 Portkey 发送您的请求可确保默认情况下所有请求都被记录
- 每个请求日志包含 `timestamp`、`model name`、`total cost`、`request time`、`request json`、`response json` 以及额外的 Portkey 功能

**[追踪](https://portkey.ai/docs/product/observability-modern-monitoring-for-llms/traces)**

- 追踪 ID 随每个请求一起传递，并在 Portkey 仪表板的日志中可见
- 如果您愿意，也可以为每个请求设置**不同的追踪 ID**
- 您还可以将用户反馈附加到追踪 ID。[此处有更多相关信息](https://portkey.ai/docs/product/observability-modern-monitoring-for-llms/feedback)

## 高级 LLMOps 功能 - 缓存、标记、重试

除了日志记录和追踪之外，Portkey 还提供了更多功能，为您的现有工作流添加生产级能力：

**缓存**

从缓存响应之前服务过的客户查询，而不是再次将它们发送到 OpenAI。匹配完全相同的字符串或语义相似的字符串。缓存可以节省成本并将延迟降低 20 倍。[文档](https://portkey.ai/docs/product/ai-gateway-streamline-llm-integrations/cache-simple-and-semantic)

**重试**

自动重新处理任何失败的 API 请求**最多 `5`** 次。使用**`指数退避`**策略，以间隔重试尝试来防止网络过载。[文档](https://portkey.ai/docs/product/ai-gateway-streamline-llm-integrations)

**标记**

使用预定义的标签详细跟踪和审计每个用户交互。[文档](https://portkey.ai/docs/product/observability-modern-monitoring-for-llms/metadata)
