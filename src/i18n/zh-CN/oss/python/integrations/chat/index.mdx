[聊天模型](/oss/langchain/models) 是一种语言模型，它以一系列[消息](/oss/langchain/messages)作为输入，并返回消息作为输出 <Tooltip tip="较旧的模型不遵循聊天模型接口，而是使用以字符串作为输入并返回字符串作为输出的接口。这些模型通常在其名称中不包含前缀 'Chat' 或包含后缀 'LLM'。">（与传统的纯文本 LLM 相对）</Tooltip>。

### 路由器和代理

路由器和代理让您可以通过单一 API 和凭证访问来自多个提供商的模型。它们可以简化计费，让您无需更改集成即可在不同模型之间切换，并提供自动回退等功能。

| 提供商 | 集成 | 描述 |
|-|-|-|
| [OpenRouter](https://openrouter.ai/) | [`ChatOpenRouter`](/oss/integrations/chat/openrouter) | 统一访问来自 OpenAI、Anthropic、Google、Meta 等公司的模型 |
| [LiteLLM](https://www.litellm.ai/) | [`ChatLiteLLM`](/oss/integrations/chat/litellm) | 为 OpenAI、Anthropic、Azure、Hugging Face 等提供统一接口，支持路由和回退 |

## 聊天补全 API

某些模型提供商提供与 OpenAI 的[聊天补全 API](https://platform.openai.com/docs/api-reference/chat) 兼容的端点。在这种情况下，您可以使用带有自定义 `base_url` 的 [`ChatOpenAI`](/oss/integrations/chat/openai) 来连接到这些端点，以实现基本的聊天功能。

<Warning>
    `ChatOpenAI` 仅针对[官方的 OpenAI API 规范](https://github.com/openai/openai-openapi)。来自第三方提供商的非标准响应字段（例如 `reasoning_content`、`reasoning`、`reasoning_details`）**不会被提取或保留**。当您需要访问非标准功能时，请使用特定于提供商的包。

    例如，OpenRouter 有专门的 LangChain 集成。有关设置和用法，请参阅 [`ChatOpenRouter` 指南](/oss/integrations/chat/openrouter)。
</Warning>
