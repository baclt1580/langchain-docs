详细文档请参阅 [OCI Generative AI 文档](https://docs.oracle.com/en-us/iaas/Content/generative-ai/home.htm) 和 [API 参考](https://docs.oracle.com/en-us/iaas/api/#/en/generative-ai/20231130/)。

## 实例化

```python
from langchain_oci import ChatOCIGenAI

llm = ChatOCIGenAI(
    model_id="meta.llama-3.3-70b-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
    model_kwargs={"temperature": 0.7, "max_tokens": 500},  # 可选
)
```

**关键参数：**
- `model_id` - 要使用的模型（参见[可用模型](#可用模型)）
- `service_endpoint` - 区域端点（`us-chicago-1`、`eu-frankfurt-1` 等）
- `compartment_id` - 您的 OCI 区间 OCID
- `model_kwargs` - 模型设置，如温度、最大令牌数

## 调用

```python
messages = [
    ("system", "你是一个代码审查助手。"),
    ("human", """审查此 Python 函数的安全问题：

def login(username, password):
    query = f"SELECT * FROM users WHERE name='{username}' AND pass='{password}'"
    return db.execute(query)
"""),
]
response = llm.invoke(messages)
print(response.content)
```

```text
此函数存在严重的 SQL 注入漏洞。用户名和密码直接插入到 SQL 查询字符串中，允许攻击者绕过身份验证或提取数据。请改用参数化查询：

    cursor.execute("SELECT * FROM users WHERE name=? AND pass=?", (username, password))
```

**多轮对话** 在消息之间保持上下文：

```python
messages = [
    ("user", "分析 14:30 UTC 的错误率峰值"),
    ("assistant", "该峰值与部署 v2.1.3 相关。正在检查日志..."),
    ("user", "根本原因是什么？"),
]
response = llm.invoke(messages)
# 模型引用了之前关于 deploy-v2.1.3 的上下文
```

## 流式传输

在生成时获取响应：

```python
for chunk in llm.stream("用 3 句话解释 Python 生成器"):
    print(chunk.content, end="", flush=True)
```

## 异步

并发处理多个请求以获得更好的吞吐量：

```python
import asyncio

# 并发分析多个代码文件
async def analyze_codebase(files: list[str]) -> list:
    tasks = [llm.ainvoke(f"查找以下代码中的漏洞：\n{code}") for code in files]
    return await asyncio.gather(*tasks)

# 流式传输响应以实现实时 UI 更新
async def stream_response():
    async for chunk in llm.astream("解释 Python 中的 async/await"):
        print(chunk.content, end="", flush=True)

asyncio.run(stream_response())
```

## 工具调用

为模型提供访问 API、数据库和自定义函数的能力：

```python
from langchain.tools import tool

@tool
def get_order_status(order_id: str) -> dict:
    """检查客户订单的状态。

    参数：
        order_id: 要查找的订单 ID
    """
    # 在生产环境中，查询您的数据库
    return {"order_id": order_id, "status": "已发货", "eta": "2024-03-15"}

@tool
def get_account_balance(account_id: str) -> dict:
    """获取当前账户余额。

    参数：
        account_id: 账户 ID
    """
    return {"account_id": account_id, "balance": 1250.00, "currency": "USD"}

# 将工具绑定到模型
tools = [get_order_status, get_account_balance]
llm_with_tools = llm.bind_tools(tools)

# 模型决定调用哪个工具
response = llm_with_tools.invoke("订单 ORD-12345 的状态是什么？")

# 检查模型是否要调用工具
if response.tool_calls:
    tool_call = response.tool_calls[0]
    print(f"工具：{tool_call['name']}, 参数：{tool_call['args']}")
    # 输出：工具：get_order_status, 参数：{'order_id': 'ORD-12345'}
```

**完整的工具执行循环** - 执行工具并返回结果：

```python
from langchain.messages import HumanMessage, AIMessage, ToolMessage

messages = [HumanMessage(content="订单 ORD-12345 的状态是什么？")]
response = llm_with_tools.invoke(messages)

# 执行每个工具调用并收集结果
if response.tool_calls:
    messages.append(response)  # 添加包含工具调用的 AI 响应

    for tool_call in response.tool_calls:
        # 查找并执行工具
        tool_fn = {"get_order_status": get_order_status,
                   "get_account_balance": get_account_balance}[tool_call["name"]]
        result = tool_fn.invoke(tool_call["args"])

        # 将工具结果添加到消息中
        messages.append(ToolMessage(content=str(result), tool_call_id=tool_call["id"]))

    # 获取包含工具结果的最终响应
    final_response = llm_with_tools.invoke(messages)
    print(final_response.content)
    # 输出：订单 ORD-12345 已发货，预计于 2024 年 3 月 15 日送达。
```

**并行工具执行**（Llama 4+）用于并发 API 调用：

```python
llm = ChatOCIGenAI(
    model_id="meta.llama-4-scout-17b-16e-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
)
llm_with_tools = llm.bind_tools(tools, parallel_tool_calls=True)
# 模型可以同时调用多个工具，减少延迟
```

## 结构化输出

将非结构化文本解析为类型化数据结构以便处理：

```python
from pydantic import BaseModel, Field
from typing import List, Literal

class SupportTicket(BaseModel):
    """客户支持工单的结构化表示。"""
    ticket_id: str
    severity: Literal["low", "medium", "high", "critical"]
    category: str = Field(description="例如：计费、技术、账户")
    description: str
    affected_services: List[str]

structured_llm = llm.with_structured_output(SupportTicket)

# 解析非结构化支持邮件
email_text = """发件人：customer@example.com
主题：紧急 - 无法访问生产数据库

我们的生产 API 在过去一小时内一直返回 500 错误。
数据库连接池似乎已耗尽。这影响了
我们的支付处理和用户身份验证服务。"""

ticket = structured_llm.invoke(email_text)
```

```python
print(ticket.severity)           # "critical"
print(ticket.category)           # "technical"
print(ticket.affected_services)  # ["payment processing", "user authentication"]
```

用于日志解析、发票提取或数据分类管道。

## 视觉与多模态

处理图像以进行数据提取、分析和自动化：

```python
from langchain.messages import HumanMessage
from langchain_oci import ChatOCIGenAI, load_image

llm = ChatOCIGenAI(
    model_id="meta.llama-3.2-90b-vision-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
)

# 分析架构图
message = HumanMessage(content=[
    {"type": "text", "text": "列出此图中的所有服务及其连接。"},
    load_image("./architecture_diagram.png"),  # 本地文件或 URL
])
response = llm.invoke([message])
print(response.content)
```

```text
该图显示了 4 个服务：
1. API 网关 - 接收外部流量，路由到内部服务
2. 身份验证服务 - 处理身份验证，连接到用户数据库
3. 订单服务 - 处理订单，连接到订单数据库和支付 API
4. 通知服务 - 发送电子邮件/SMS，由订单服务触发
```

**用例：** 图表分析、收据/发票解析、图表数据提取、文档处理

**视觉模型：** Llama 3.2 Vision (11B, 90B), Gemini 2.0/2.5, Grok 4, Cohere Command A

## Gemini 多模态（PDF、视频、音频）

使用 Gemini 模型处理文档、视频和音频：

```python
import base64
from langchain.messages import HumanMessage
from langchain_oci import ChatOCIGenAI

llm = ChatOCIGenAI(
    model_id="google.gemini-2.5-flash",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
)

# 从 PDF 中提取数据
with open("contract.pdf", "rb") as f:
    pdf_data = base64.b64encode(f.read()).decode()

message = HumanMessage(content=[
    {"type": "text", "text": "以 JSON 格式提取合同方、生效日期和付款条款。"},
    {"type": "document_url", "document_url": {"url": f"data:application/pdf;base64,{pdf_data}"}}
])
response = llm.invoke([message])
print(response.content)
```

```text
{
  "parties": ["Acme Corp", "TechStart Inc"],
  "effective_date": "2024-01-15",
  "payment_terms": "Net 30, monthly invoicing"
}
```

**视频/音频分析：**

```python
with open("meeting.mp4", "rb") as f:
    video_data = base64.b64encode(f.read()).decode()

message = HumanMessage(content=[
    {"type": "text", "text": "列出行动项以及每项的负责人。"},
    {"type": "video_url", "video_url": {"url": f"data:video/mp4;base64,{video_data}"}}
])
response = llm.invoke([message])
```

**支持的格式：** PDF、MP4/MOV 视频、MP3/WAV 音频（仅限 Gemini 2.0/2.5）

## 配置

使用 `model_kwargs` 控制模型行为：

```python
llm = ChatOCIGenAI(
    model_id="meta.llama-3.3-70b-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
    model_kwargs={
        "temperature": 0.7,    # 创造性：0 = 确定性，1 = 创造性
        "max_tokens": 500,     # 最大响应长度
        "top_p": 0.9,          # 核心采样阈值
    },
)
```
