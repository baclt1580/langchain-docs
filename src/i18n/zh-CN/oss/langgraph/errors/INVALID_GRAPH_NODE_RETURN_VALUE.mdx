---
title: INVALID_GRAPH_NODE_RETURN_VALUE
---

:::python
LangGraph 的 [`StateGraph`](https://langchain-ai.github.io/langgraph/reference/graphs/#langgraph.graph.state.StateGraph) 收到了来自节点的非字典类型返回值。以下是一个示例：

```python
class State(TypedDict):
    some_key: str

def bad_node(state: State):
    # 应返回包含 "some_key" 值的字典，而不是列表
    return ["whoops"]

builder = StateGraph(State)
builder.add_node(bad_node)
...

graph = builder.compile()
```

调用上述图将导致如下错误：

```python
graph.invoke({ "some_key": "someval" });
```

```
InvalidUpdateError: Expected dict, got ['whoops']
如需故障排除，请访问：https://docs.langchain.com/oss/python/langgraph/errors/INVALID_GRAPH_NODE_RETURN_VALUE
```

图中的节点必须返回一个字典，其中包含在状态中定义的一个或多个键。
:::

:::js
LangGraph 的 [`StateGraph`](https://langchain-ai.github.io/langgraph/reference/graphs/#langgraph.graph.state.StateGraph) 收到了来自节点的非对象类型返回值。以下是一个示例：

```typescript
import { z } from "zod/v4";
import { StateGraph, StateSchema, GraphNode } from "@langchain/langgraph";

const State = new StateSchema({
  someKey: z.string(),
});

const badNode: GraphNode<typeof State> = (state) => {
  // 应返回包含 "someKey" 值的对象，而不是数组
  return ["whoops"];
};

const builder = new StateGraph(State).addNode("badNode", badNode);
// ...

const graph = builder.compile();
```

调用上述图将导致如下错误：

```typescript
await graph.invoke({ someKey: "someval" });
```

```
InvalidUpdateError: Expected object, got ['whoops']
如需故障排除，请访问：https://langchain-ai.github.io/langgraphjs/troubleshooting/errors/INVALID_GRAPH_NODE_RETURN_VALUE
```

图中的节点必须返回一个对象，其中包含在状态中定义的一个或多个键。
:::

## 故障排除

以下方法可能有助于解决此错误：

:::python
* 如果节点中包含复杂逻辑，请确保所有代码路径都返回适合已定义状态的字典。
:::

:::js
* 如果节点中包含复杂逻辑，请确保所有代码路径都返回适合已定义状态的对象。
:::
