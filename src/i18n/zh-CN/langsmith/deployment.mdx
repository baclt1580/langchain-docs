LangSmith 部署是一个专为智能体工作负载设计的工作流编排运行时。它提供了智能体在生产环境中大规模可靠运行所需的托管基础设施，支持从本地开发到部署的完整生命周期。LangSmith 部署是框架无关的：您可以部署使用 LangGraph 或[其他框架](/langsmith/deploy-other-frameworks)构建的智能体。

<Note>
LangSmith 部署需要 [Plus 计划或更高版本](https://www.langchain.com/pricing)。
</Note>

## 能力

### Studio

[Studio](/langsmith/studio) 可以连接到任何智能体服务器（本地或已部署的），并为您提供一个用于开发和调试智能体的交互式环境。可视化执行图、在任何检查点检查状态、单步执行运行、在执行过程中修改状态，以及分支以探索替代路径。

### 智能体组合

智能体并非孤立运行。[RemoteGraph](/langsmith/use-remote-graph) 允许任何智能体调用其他已部署的智能体，使用与本地相同的接口：研究智能体委托给不同部署上的搜索智能体，路由智能体分派给专门的子智能体。智能体无需知道它们调用的是本地还是远程服务。

对 [MCP](/langsmith/server-mcp) 和 [A2A](/langsmith/server-a2a) 的原生支持意味着您部署的智能体可以暴露和使用工具接口以及智能体间协议，并与更广泛的生态系统集成。

### 部署选项

- **[云端](/langsmith/deploy-to-cloud)**：完全托管。从 git 仓库推送或使用 [`langgraph deploy`](/langsmith/cli#deploy)。
- **[混合](/langsmith/deploy-with-control-plane)**：在您的云中运行，由 LangSmith [控制平面](/langsmith/control-plane) 管理。
- **[自托管](/langsmith/self-hosted)**：在您自己的基础设施中完全自我管理。

相同的运行时，相同的 API。变化的是谁管理基础设施。有关比较，请参阅 [平台设置](/langsmith/platform-setup)。
