## 设置与安装

要使用此功能，请安装 `langchain-hana` 包：

```python
pip install langchain_hana
```

然后，创建到您的 SAP HANA Cloud 实例的连接。

```python
import os

from dotenv import load_dotenv
from hdbcli import dbapi

# 如果需要，加载环境变量
load_dotenv()

# 建立到 SAP HANA Cloud 的连接
connection = dbapi.connect(
    address=os.environ.get("HANA_DB_ADDRESS"),
    port=os.environ.get("HANA_DB_PORT"),
    user=os.environ.get("HANA_DB_USER"),
    password=os.environ.get("HANA_DB_PASSWORD")
)
```

`HanaSparqlQAChain` 将以下部分整合在一起：

1.  **支持模式的 SPARQL 生成**
2.  针对 SAP HANA 的**查询执行**
3.  **自然语言答案格式化**

## 初始化

您需要：

*   一个用于生成和解释查询的 **LLM**
*   一个 **`HanaRdfGraph`**（包含连接、`graph_uri` 和本体）

请按照 [HanaRdfGraph](/oss/python/integrations/graphs/sap_hana_rdf_graph) 中的步骤了解更多关于创建 `HanaRdfGraph` 实例的信息。

导入 `HanaSparqlQAChain`

```python
from langchain_hana import HanaSparqlQAChain
```

```python
qa_chain = HanaSparqlQAChain.from_llm(
    llm=llm, graph=graph, allow_dangerous_requests=True, verbose=True
)
```

## 流程概述

1.  **SPARQL 生成**
    *   使用 `SPARQL_GENERATION_SELECT_PROMPT`
    *   输入：
        *   `schema`（来自 `graph.get_schema` 的 Turtle 格式）
        *   `prompt`（用户的问题）
2.  **查询后处理**
    *   从 LLM 输出中提取 SPARQL 代码。
    *   如果缺失，则注入 `FROM <graph_uri>`
    *   确保声明了必需的通用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）
3.  **执行**
    *   调用 `graph.query(generated_sparql)`
4.  **答案构建**
    *   使用 `SPARQL_QA_PROMPT`
    *   输入：
        *   `context`（原始查询结果）
        *   `prompt`（原始问题）

## 提示模板

### "SPARQL 生成" 提示

`sparql_generation_prompt` 用于引导 LLM 根据用户问题和提供的模式生成 SPARQL 查询。

### 答案生成提示

`qa_prompt` 指示 LLM 仅基于数据库结果创建自然语言答案。

默认提示可以在这里找到：[`prompts.py`](https://github.com/SAP/langchain-integration-for-sap-hana-cloud/blob/main/langchain_hana/chains/graph_qa/prompts.py)

## 自定义提示

您可以在初始化时覆盖默认值：

```python
qa_chain = HanaSparqlQAChain.from_llm(
    llm=llm,
    graph=graph,
    allow_dangerous_requests=True,
    verbose=True,
    sparql_generation_prompt=YOUR_SPARQL_PROMPT,
    qa_prompt=YOUR_QA_PROMPT
)
```

> - `sparql_generation_prompt` 必须包含输入变量：`["schema", "prompt"]`
> - `qa_prompt` 必须包含输入变量：`["context", "prompt"]`

## 示例：在“电影”知识图谱上进行问答

**前提条件**：
您必须拥有一个启用了 **三元组存储** 功能的 SAP HANA Cloud 实例。
详细说明请参考：[启用三元组存储](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/enable-triple-store/)<br />
加载 `kgdocu_movies` 示例数据。请参阅 [知识图谱示例](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/knowledge-graph-example)。

下面我们将：

1.  实例化指向我们“电影”数据图的 `HanaRdfGraph`
2.  将其包装在由 LLM 驱动的 `HanaSparqlQAChain` 中
3.  提出自然语言问题并打印出链的响应

这演示了 LLM 如何在后台生成 SPARQL，针对 SAP HANA 执行它，并返回人类可读的答案。

首先，创建到您的 SAP HANA Cloud 实例的连接。

```python
import os

    address=os.environ.get("HANA_DB_ADDRESS"),

    user=os.environ.get("HANA_DB_USER"),
    password=os.environ.get("HANA_DB_PASSWORD")
```

## 幕后发生了什么？

1.  **SPARQL 生成**
    该链使用 `SPARQL_GENERATION_SELECT_PROMPT`，调用 LLM 并传入您的 Turtle 格式本体（`graph.get_schema`）和用户的问题。然后 LLM 生成一个针对您模式定制的有效 `SELECT` 查询。
2.  **预处理与执行**
    *   **提取与清理**：从 LLM 的响应中提取原始 SPARQL 文本。
    *   **注入图谱上下文**：如果缺失，则添加 `FROM <graph_uri>`，并确保声明了通用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）。
    *   **在 HANA 上运行**：通过 `HanaRdfGraph.query()` 在您的命名图上执行最终确定的查询。
3.  **答案构建**
    返回的 CSV（或 Turtle）结果再次输入到 LLM 中——这次使用 `SPARQL_QA_PROMPT`。LLM 根据检索到的数据生成一个简洁、人类可读的答案，严格基于数据，不会产生幻觉。
